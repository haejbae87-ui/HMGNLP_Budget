# 예산/정산 표준 용어 및 공식 사전

> **문서 목적**
> HMG 교육 예산 시스템의 잦은 명칭 혼동을 방지하고, 예산의 배분부터 집행, 잔액 확인까지의 **추적성(Traceability)**을 시스템적으로 보장하기 위해 정의된 **단일 진실 공급원(SSOT)**입니다.
> 시스템 기획, UI 개발, 정산 로직 구현 시 **반드시 이 문서의 명칭과 공식을 기준**으로 삼아야 합니다.

---

## 1. 6대 예산 표준 용어

| 분류 | 표준 명칭 | 영문 맵핑(DB) | 설명 및 성격 |
|---|---|---|---|
| **계획** | **사업계획금액** | `requested_budget` | 수요예측(사업계획) 단계에서 최초 요청한 기준 금액 (Baseline) |
| **계획** | **운영계획금액** | `amount` | 팀에 배정받은 예산 범위 내에서 실제 운영을 위해 편성한 금액 |
| **통장** | **최초배정액** | `allocated_amount` | BO 총괄담당자가 각 팀의 가상 예산 통장에 최초로 입금해준 실제 가용 자본금 |
| **지출** | **약정금액** | `hold_amount` | 교육 신청 후 결재가 진행 중이어서 타 용도로 쓸 수 없게 임시로 묶인(Hold) 돈 |
| **지출** | **집행금액** | `actual_amount` | 결재(품의)가 최종 완료되어 통장에서 실제로 차감(사용) 확정된 돈 |
| **잔액** | **가용예산** | `available_amount` | 통장에 배정된 총액 중, 아직 쓰지 않고 남아있어 다른 교육에 추가 배정/사용이 가능한 돈 |

## 2. 예산 공식 (Traceability Formula)

예산 시스템 무결성을 위해, 시스템 내의 잔액은 절대 하드코딩(수기 저장)되어선 안 되며, 다음 공식에 의해 항상 동적으로 계산되어야 합니다.

> **가용예산 = 최초배정액 - (약정금액 + 집행금액)**

### 공식 검증 시나리오
1. **최초 배정 시점**: 약정=0, 집행=0 ➔ `가용예산 = 최초배정액`
2. **교육 신청 시 (결재중)**: 결재 대기 상태이므로 약정액 증가 ➔ `가용예산 감소`
3. **결재 반려/취소 시**: 약정액 감소(원복) ➔ `가용예산 복구`
4. **결재 승인(완료) 시**: 약정액 감소, 동시에 집행액 증가 ➔ `가용예산 유지 (이미 감소분 반영됨)`

## 3. UI 설계 가이드라인 (Naming Convention)

FO/BO 모든 화면의 라벨, 뱃지, 엑셀 다운로드 컬럼명은 다음 원칙을 따릅니다.
- `배정금액`, `배정액` ➔ **최초배정액**
- `점유액`, `홀딩금액` ➔ **약정금액**
- `사용액`, `실지출액` ➔ **집행금액**
- `잔여액`, `남은 돈` ➔ **가용예산**
