# SC-001: 승인된 교육계획 예산 재배치

> **제안일**: 2026-04-19
> **상태**: 🟡 설계 완료
> **업무 목적**: 승인받은 교육계획의 배정 예산을 줄이고, 남는 예산으로 다른 교육계획을 수립·승인·신청할 수 있어야 한다
> **관련 PRD**: [fo_submission_approval.md](../PRD/fo_submission_approval.md), [budget_lifecycle.md](../PRD/budget_lifecycle.md)

---

## 시나리오 스텝

1. **FO 사용자**: "파이썬 교육" 계획 수립 (계획액: 100만원)
2. **상신** → 운영담당자 1차 검토 → 총괄담당자 최종 승인
3. **총괄**: 배정액 80만원으로 조정 (plans.allocated_amount = 800,000)
4. ── 시간 경과 (운영 환경 변화) ──
5. **FO 사용자**: "파이썬보다 클로드코드가 더 중요하다" 판단
6. **FO 사용자**: 파이썬 교육 배정액 80만원 → **0원으로 축소** (예산 환불)
7. **FO 사용자**: "클로드코드" 교육계획 80만원 신규 수립
8. **상신** → 승인
9. **FO 사용자**: 클로드코드 승인된 계획 기반으로 **교육 신청**

## 대안 경로

> 사용자가 원래 시나리오(Step 6)에서 "배정액을 0원으로 직접 변경"하는 대신,
> **"📉 배정액 축소" 기능**을 통해 80만→0원으로 축소하면 통장에 80만원이 자동 환불됨.
> 이후 클로드코드 계획 수립→상신→승인→신청까지 정상 진행 가능.

## 검증 체크리스트

- [ ] **Step 1~3**: 파이썬 교육 계획 수립 → 상신 → 승인 (allocated_amount = 800,000)
- [ ] **Step 6a**: 파이썬 교육 상세에서 "📉 배정액 축소" 버튼 존재 확인
- [ ] **Step 6b**: 배정액 80만 → 0원 축소 시 `used_amount -= 800,000` 환불 확인
- [ ] **Step 6c**: 축소 후 교육명 등 내용 필드 편집 불가 확인
- [ ] **Step 6d**: 통장 잔액에 80만원 복원 확인
- [ ] **Step 7**: 클로드코드 계획 80만원 수립 시 잔액 검증 통과 확인
- [ ] **Step 8**: 클로드코드 상신 → Hold 80만원 → 승인 → used_amount +80만원
- [ ] **Step 9**: 클로드코드 계획 기반 교육신청(application) 생성 가능 확인
- [ ] **EC-P01**: 이미 교육신청이 연결된 계획에서 축소 시도 → 신청 금액 이하로 축소 차단
- [ ] **EC-P03**: 부분 축소 (80만→50만) → 30만원 환불, 50만원 유지 확인
