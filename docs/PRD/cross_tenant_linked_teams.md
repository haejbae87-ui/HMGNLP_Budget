# PRD: 총괄부서 크로스 테넌트 연동

> **문서 ID**: PRD-CT-001  
> **버전**: 1.0  
> **작성일**: 2026-04-10  
> **상태**: 검토 요청  

---

## 1. 배경 및 목적

### 1.1 비즈니스 배경

현대자동차(HMC)와 기아(KIA)에는 **총괄부서**와 **직속부서** 두 가지 유형의 조직이 존재한다.

| 구분 | 총괄부서 | 직속부서 |
|---|---|---|
| 존재 범위 | HMC + KIA **양쪽 모두 존재** | 각 회사에만 독립 존재 |
| 조직코드 | HMC/KIA **동일 코드** (회사코드만 다름) | 각 회사 고유 코드 |
| 조직장 소속 | HMC 소속 **또는** KIA 소속 (유동적) | 해당 회사 소속 |
| 데이터 공유 | 양사 데이터 **상호 조회 가능** | 해당 회사 데이터만 |
| 예: | 역량혁신팀, (기타 다수) | 내구기술팀(HMC), 개인정보보호팀(KIA) |

### 1.2 핵심 요구사항

> **"회사코드가 다르지만 조직코드가 같으면, 같은 데이터를 볼 수 있어야 한다"**

총괄부서 소속 직원은 **양쪽 회사의 교육계획, 교육신청, 교육결과, 예산을 동등하게 조회·사용**할 수 있어야 한다.

## 3. 요구사항 정의

### 3.1 기능 요구사항 (FR)

#### FR-01: 교육계획 크로스 테넌트 공유
- **설명**: 총괄부서 소속 직원이 양쪽 회사의 교육계획을 조회할 수 있어야 한다.
- **예시**: HMC 역량혁신팀 최O영이 수립한 교육계획을, KIA 역량혁신팀 정O식도 조회 가능
- **우선순위**: P0 (필수)

#### FR-02: 크로스 테넌트 교육신청
- **설명**: 총괄부서 팀원이 상대 회사 팀원이 수립한 교육계획을 기반으로 교육신청이 가능해야 한다.
- **예시**: HMC 최O영이 수립한 계획으로 KIA 정O식이 교육신청 가능
- **조건**: 각 서비스 정책에 맵핑된 계정의 팀별 예산 통장에서 예산 차감
- **우선순위**: P0 (필수)

#### FR-03: 크로스 테넌트 예산 조회
- **설명**: 총괄부서 소속 직원이 양쪽 회사의 예산 상세 배정 및 사용 현황을 조회할 수 있어야 한다.
- **범위**: 잔액 + 상세 배정 + 사용 내역 (Full 공유)
- **방향**: 양방향 (HMC↔KIA)
- **우선순위**: P0 (필수)

#### FR-04: 크로스 테넌트 예산 사용
- **설명**: HMC 총괄부서 팀원이 KIA 예산을 사용(차감)할 수 있고, 그 반대도 가능해야 한다.
- **메커니즘**: 양쪽 VOrg에 총괄조직을 수동 맵핑 → 서비스 정책으로 계정 노출 → 해당 계정의 팀별 통장에서 차감
- **우선순위**: P0 (필수)

#### FR-05: 크로스 테넌트 결재
- **설명**: 총괄부서 팀장(조직장)이 상대 회사에서 상신된 교육계획·신청·결과에 대한 결재를 처리할 수 있어야 한다.
- **조건**: 팀장만 결재 가능 (팀원 불가)
- **결재 후 예산**: 각 서비스 정책에 맵핑된 계정의 팀별 예산 통장에서 차감
- **우선순위**: P0 (필수)

#### FR-06: BO 연결 조직 관리
- **설명**: 백오피스에서 총괄부서(연결 조직)를 관리할 수 있어야 한다.
- **기능**: 
  - 조직 등록 시 "총괄부서(linked)" 체크 → HMC/KIA 양쪽에 같은 레벨·같은 위치에 자동 생성
  - 연결 해제/수정 UI
- **우선순위**: P1 (중요)

#### FR-07: VOrg 수동 맵핑
- **설명**: 총괄부서를 HMC 가상조직템플릿과 KIA 가상조직템플릿에 각각 수동으로 맵핑한다.
- **이유**: 자동 맵핑은 사용자가 모르는 오류를 유발할 수 있음
- **결과**: 양쪽 VOrg에 맵핑된 총괄부서는 각각의 예산계정을 바라볼 수 있음
- **우선순위**: P0 (필수, 운영 가이드)

### 3.2 비기능 요구사항 (NFR)

#### NFR-01: 적용 범위 제한
- **HMC↔KIA 간에만** 총괄부서 연동 적용
- 현대오토에버(HAE), 현대제철(HSC) 등은 영향 없음
- 직속부서는 기존 방식 유지

#### NFR-02: 대칭 권한
- HMC→KIA, KIA→HMC 동일한 권한 (비대칭 없음)

#### NFR-03: 데이터 보안
- Full access (상세 정보 포함) 기본 적용
- BO에서 linked 설정 변경으로 해제 가능

### 3.3 명시적 제외 사항

| 항목 | 설명 |
|---|---|
| 현대제철 연동 | HMC↔KIA만 해당. HSC 등 타 계열사 미적용 |
| 자동 VOrg 맵핑 | 사용자 오류 방지를 위해 수동 맵핑 |
| KIA 데모 페르소나 | 필요 시 DB 직접 INSERT (자동 생성 불필요) |

## 4. 핵심 시나리오

### 시나리오 A: 교육계획 크로스 공유

```
1. HMC 역량혁신팀 최O영이 교육계획 수립 (tenant_id: HMC)
2. KIA 역량혁신팀 정O식이 FO 교육계획 목록 접속
3. 시스템: 정O식의 org_code → 같은 org_code의 HMC 조직 검색 → 존재 (총괄부서)
4. 시스템: plans WHERE tenant_id IN ('KIA', 'HMC') AND 조직 매칭
5. 화면: KIA 계획 + HMC 계획(🏷️[현대] 뱃지) 통합 표시
```

### 시나리오 B: 크로스 테넌트 교육신청

```
1. HMC 최O영이 "HMGNLP 설계 세미나" 교육계획 수립 → 승인
2. KIA 정O식이 교육 신청 화면 접속
3. 승인된 교육계획 선택 → HMC 최O영의 계획도 목록에 표시
4. 정O식이 HMC 최O영의 계획을 선택하여 교육신청 작성
5. 예산 차감: 정O식의 KIA 예산 통장에서 차감 (서비스 정책 맵핑 기준)
```

### 시나리오 C: 크로스 테넌트 결재

```
1. KIA 정O식이 교육계획 상신 (tenant_id: KIA)
2. HMC 역량혁신팀 팀장(조직장)이 리더 결재함 접속
3. 시스템: 팀장의 org_code → linked KIA 조직 검색 → KIA pending 문서 포함
4. 팀장: KIA 문서(🏷️[기아] 뱃지) 확인 후 승인
5. 예산: KIA 서비스 정책 맵핑 계정의 KIA 통장에서 차감
```

## 5. 데이터 모델 변경

### 5.1 organizations 테이블 확장

```sql
ALTER TABLE organizations 
  ADD COLUMN org_code text DEFAULT NULL,       -- 조직코드 (회사 간 동일)
  ADD COLUMN org_type text DEFAULT 'direct';   -- 'general'(총괄) | 'direct'(직속)
```

### 5.2 연결 매칭 방식

기존 `linked_teams` 테이블 대신 **조직코드 기반 자동 매칭**:

```
총괄부서 판별 규칙:
1. organizations.org_type = 'general'
2. HMC에 org_code='ABC', KIA에도 org_code='ABC' → 연결 확인
3. 쿼리 시: WHERE org_code = 내_org_code AND tenant_id IN ('HMC','KIA')
```

> **장점**: 별도 linked_teams 테이블 불필요. 조직코드만으로 자동 매칭.  
> **관리**: org_type='general' 설정만으로 총괄부서 활성화/비활성화 가능.

## 6. UI 변경 사항

### 6.1 FO 교육계획·신청·결과 목록

- 상대 회사 데이터에 **`[현대]` / `[기아]` 뱃지** 표시
- 자사 데이터와 상대사 데이터 시각적 구분 (뱃지 색상)
- 상대사 데이터도 상세보기 가능

### 6.2 FO 리더 결재함

- 상대 회사 pending 문서도 표시 (회사 뱃지 포함)
- 결재 처리 동일 (승인/반려)

### 6.3 BO 조직 관리

- 조직 등록/수정 시 "총괄부서" 체크박스
- 총괄부서 체크 시 → `org_code` 입력란 표시
- "이 조직코드와 매칭되는 상대 회사 조직" 미리보기

### 6.4 BO VOrg 가상조직 매핑

- 총괄부서를 HMC VOrg, KIA VOrg에 **각각 수동 맵핑** (기존 방식 유지)
- 맵핑 시 뱃지로 "🔗 총괄부서" 표시
