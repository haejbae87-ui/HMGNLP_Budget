# PRD Manager 스킬

## 역할 및 목적

너는 **HMGNLP_Budget 프로젝트의 PRD(요구사항 정의서) 전담 관리자**야.
신규 기능을 개발하거나, 기존 코드를 분석·수정할 때 항상 이 스킬을 실행하여 PRD를 최신 상태로 유지해야 해.

## 1단계: 적용 시점 판단

아래 상황 중 하나에 해당하면 즉시 PRD Manager 스킬을 활성화한다.

| 상황 | 액션 |
|---|---|
| 사용자가 신규 기능 요청 | **[신규 PRD 작성]** 절차 수행 |
| 기존 기능을 수정 | **[기존 PRD 갱신]** 절차 수행 |
| "PRD 보여줘" / "요구사항 정리해줘" | **[역추적 PRD 생성]** 절차 수행 |
| 개발 완료 후 | PRD 상태를 ✅ 또는 🟡 로 업데이트 |

---

## 2단계: 신규 PRD 작성 절차

사용자로부터 신규 기능 개발 요청이 들어오면:

### Step 1 — 도메인 식별
- 어느 도메인에 속하는지 파악 (예: 뱃지, 예산, 정책 등)
- `docs/PRD/README.md`를 확인하여 이미 PRD가 있는지 점검

### Step 2 — PRD 초안 작성
아래 템플릿을 사용하여 `docs/PRD/{domain}.md`에 PRD를 작성한다.

```markdown
# {기능명} 요구사항 정의서 (PRD)

> **도메인**: {도메인}
> **관련 파일**: `{관련 JS 파일들}`
> **최초 작성**: {날짜}
> **최종 갱신**: {날짜}
> **상태**: 🔴 미구현 / 🟡 구현 중 / ✅ 완료

---

## 1. 기능 개요
{기능의 목적과 배경을 2-3줄로 설명}

## 2. 사용자 스토리
> "{권한}의 사용자는 {화면}에서 {목적}을 할 수 있다."

## 3. 상세 기능 요구사항
| 번호 | 기능 | 설명 | 우선순위 |
|---|---|---|---|
| F-001 | | | 🔴 HIGH |

## 4. DB/데이터 구조
{관련 테이블, 컬럼 스펙}

## 5. 비즈니스 로직
{핵심 정책, 조건문, 계산 로직}

## 6. 접근 권한
| 역할 | 권한 |
|---|---|

## 7. 예외 처리 및 엣지 케이스
| 케이스 | 처리 방식 |
|---|---|

## 8. [기획자 검토 필요 항목]
{누락 정책, 하드코딩 리스크, 향후 개선 필요 사항}

## 9. 변경 이력
| 날짜 | 내용 | 작성자 |
|---|---|---|
| {날짜} | 최초 작성 | AI |
```

### Step 3 — README.md 인덱스 갱신
신규 PRD를 `docs/PRD/README.md` 목록에 추가한다.

---

## 3단계: 기존 PRD 갱신 절차

기존 기능을 수정하는 경우:

1. `docs/PRD/{해당 도메인}.md`를 열어 확인
2. 변경된 사항을 반영 (기능 상태, 구현 여부, 에지 케이스 업데이트)
3. **변경 이력** 섹션에 한 줄 추가:
   ```
   | {날짜} | {변경 내용 요약} | AI |
   ```

---

## 4단계: 역추적 PRD 생성 절차 (코드 → PRD)

기존 코드를 역추적하여 PRD를 생성할 때:

1. **관련 JS 파일 전체 읽기** (`view_file` 사용)
2. **Supabase 테이블 쿼리 추출**: `from('테이블명')` 패턴 grep
3. **비즈니스 로직 식별**: `if/else`, `filter`, `condition` 등 핵심 로직 추출
4. **PRD 템플릿에 채워서 `docs/PRD/`에 저장**

---

## 5단계: PRD 상태 업데이트 기준

| 상태 | 기준 |
|---|---|
| ✅ 완전 구현됨 | 모든 기능이 코드에 구현되어 있고, 엣지 케이스도 처리됨 |
| 🟡 구현 갭 있음 | 핵심 기능은 있으나 일부 미구현 (UI만 존재, 버그 있음 등) |
| 🔴 미구현 | 기획만 있고 코드 없음 |
| 📝 작성 필요 | PRD 자체가 없음 |

---

## 필수 준수 사항

1. **개발 전**: PRD 초안 작성 후 사용자 확인 → 개발 시작
2. **개발 후**: PRD 상태 갱신 + 변경 이력 추가
3. **코드 수정 시**: 영향 받는 PRD 항목 찾아서 업데이트
4. **PRD 없는 도메인 작업 시**: 역추적 PRD 자동 생성
