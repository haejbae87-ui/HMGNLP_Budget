# Progress Signal 스킬

## 역할 및 목적
개발 중 AI가 긴 시간(30초 이상) 생각을 하거나 복수 파일 수정, 다수의 스크립트를 실행할 때 사용자의 화면(IDE)에서는 아무 일도 안 일어나는 것처럼 보일 수 있습니다. 이로 인해 사용자가 작업을 취소하는 불상사를 방지하기 위해, **세션(대화창)이 바뀌더라도 항상 동일하게 유지되는 워크스페이스 내 파일(`docs/task.md`)을 활용하여 현재 작업 진행 상황(시그널)을 실시간으로 표시**하는 스킬입니다.

## 🚨 필수 적용 규칙 (모든 세션 공통)

1. **글로벌 Task 파일 사용 (`docs/task.md`)**:
   - AI는 복잡한 작업(예: P12 개발, 모듈 리팩토링 등)을 시작하기 직전에, 로컬 워크스페이스의 `docs/task.md` 파일을 엽니다.
   - 대화 세션에 종속되는 `<appDataDir>\brain\...\task.md` 뿐만 아니라, **반드시 `docs/task.md` 파일을 우선적으로 갱신**하여 사용자가 항상 열어두고 볼 수 있는 진행 알림판으로 사용합니다.

2. **실시간 시그널 기록 방식 (Progress Signalling)**:
   - 개발을 여러 단계로 나눌 경우, 각 단계(Step)별 코딩이나 명령어 실행 **직전**에 `docs/task.md`를 덮어쓰기(`multi_replace_file_content` 또는 `write_to_file`)하여 현재 "무엇을 하고 있는지"를 알립니다.
   - 예시:
     ```markdown
     # 🚧 실시간 AI 작업 진행 상황
     - [x] Step 1: `bo_approval.js` 분석 완료
     - [ ] ⏳ Step 2: `fo_plans_list.js` 묶음 상신 UI 구현 (현재 진행 중... 🛠️)
     - [ ] Step 3: API 연동 및 결재선 데이터 가공
     ```

3. **명령어 출력을 통한 시그널 (선택)**:
   - 터미널을 주시하는 사용자를 위해 백그라운드 명령어(`run_command`)를 날릴 때 간단한 echo 문을 포함하여 진행 상황이 터미널 창에도 출력되도록 할 수 있습니다. (예: `echo "📌 [1/3] 파일 수정 중..."`)

## 즉시 실행 지침
- 이 스킬을 읽은 AI는 **지금 즉시** `docs/task.md`를 열어 진행 상황판 형식으로 내용을 덮어쓰고, 다가올 개발(예: P12)의 스텝들을 명시하세요.
- 각 스텝을 실행하기 전에 `docs/task.md`의 `[ ]` 상태를 `[⏳ 진행 중...]`으로 변경한 후 도구(tool)를 호출해야 합니다.
