# 사업계획(수요예측) 및 운영계획(상시계획) 요구사항 정의서 (PRD)

> **도메인**: 학습 계획·신청
> **관련 파일**: `public/js/plans.js`, `public/js/approval.js`, `public/js/fo_plans_list.js`, `public/js/fo_plans_wizard.js`, `public/js/bo_plan_mgmt.js`
> **최초 작성**: 2026-04-27
> **최종 갱신**: 2026-04-27
> **상태**: 🟡 구현 중

## 1. 기능 개요
사용자의 피드백을 수용하여, 예산을 확보하기 위한 **'사업계획(구 수요예측)'**과 예산을 실제로 집행하기 위한 **'운영계획(구 상시계획)'**으로 교육계획 워크플로우를 분리합니다. 

## 2. 사용자 스토리
> "FO 팀 대표는 수요예측 기간에 작성된 팀원들의 사업계획을 묶어서 팀장에게 일괄 상신할 수 있다."
> "BO 운영담당자는 올라온 사업계획들을 보고 1차 배정액을 기입하며, BO 총괄담당자는 이를 보고 최종 배정액을 승인하여 각 팀의 통장에 예산을 충전해 준다."
> "FO 팀원은 승인되어 자동 생성된 운영계획의 배정액 한도 내에서 실제 교육을 신청할 수 있으며, 필요 시 배정액을 다른 운영계획으로 재조정할 수 있다."

## 3. 상세 기능 요구사항
| 번호 | 기능 | 설명 | 우선순위 |
|---|---|---|---|
| F-001 | 사업계획/운영계획 FO 탭 분리 | FO 교육계획 수립 화면에서 사업계획과 운영계획을 탭으로 분리하여 조회. | 🔴 HIGH |
| F-002 | 예산 계정 필터링 | FO 교육계획 화면에서 예산 계정을 먼저 선택하도록 하여 화면 복잡도 완화. | 🔴 HIGH |
| F-003 | 사업계획 고정 결재라인 | 사업계획 상신 시 정책빌더를 타지 않고 [기안자→팀장→운영담당자→총괄담당자] 고정 결재 수행. | 🔴 HIGH |
| F-004 | BO 예산 심의 (배정액 할당) | BO에서 사업계획별로 allocated_amount를 입력받아 승인 처리. | 🔴 HIGH |
| F-005 | 운영계획 자동 복사 | 사업계획이 총괄 승인되면 plan_type='operation'으로 원본을 복사하고 allocated_amount를 이관. | 🔴 HIGH |
| F-006 | 운영계획 배정 재배분 | FO에서 운영계획 간에 allocated_amount를 깎고 늘리며 재조정하는 기능. (사업계획 탭에서는 배정 재배분 UI 완전 노출 제외) | 🟡 MED |
| F-007 | 탭별 맞춤 대시보드 | 사업계획: 전체 건수(승인/반려 포함), 사업계획 금액 총액, 승인된 배정액 총액<br>운영계획: 전체, 진행중, 완료, 반려 통계 | 🔴 HIGH |

## 4. DB/데이터 구조
- **plans 테이블 추가 컬럼**:
  - `plan_type`: `business` (사업계획), `operation` (운영계획)
  - `allocated_amount` (NUMERIC): 총괄담당자가 배정한 최종 금액.
  - `parent_id` (UUID): 운영계획이 복사될 때 원본 사업계획의 ID를 저장.

## 5. 비즈니스 로직
- 사업계획 승인 완료 시, 각 사업계획의 `allocated_amount`의 총합이 `account_budgets` 테이블의 `total_amount` 또는 `available_amount`에 충전(Credit)되어야 함.
- 사업계획 승인 시, 기존 상태 전이(`approved`)와 동시에 `plans` 테이블에 동일 내용으로 `plan_type='operation'` 레코드 생성.
- FO 교육신청은 오직 `plan_type='operation'` 인 문서만 참조할 수 있음.
- FO 대시보드에서 사업계획 모드(`isBusiness`)일 때는 '배정 재배분' 버튼을 감추고 금액 기반 통계(계획 금액, 배정 금액)를 우선 표시하여 예산 가시성을 확보함.

## 6. 접근 권한
| 역할 | 권한 |
|---|---|
| FO 일반 사용자 | 본인의 사업계획/운영계획 수립 및 상신 (단건) |
| FO 팀 대표 (팀장/매니저) | 팀원 사업계획 일괄 취합 상신, 운영계획 배정액 재조정 |
| BO 운영담당자 | 사업계획 1차 배정액 기입 및 총괄 상신 |
| BO 총괄담당자 | 사업계획 최종 배정액 승인 및 통장 충전 |

## 7. 예외 처리 및 엣지 케이스
| 케이스 | 처리 방식 |
|---|---|
| 사업계획 금액을 BO에서 삭감한 경우 | FO 운영계획에는 원래 신청금액(amount)과 삭감된 배정금액(allocated_amount)이 같이 표시됨. 부족분은 다른 운영계획에서 재배분해야 함. |
| 상시 돌발 운영계획 수립 시 | 사업계획을 안 거쳤으므로 allocated_amount = 0원. 다른 운영계획의 배정액을 재조정해와야 교육신청 가능. |
| 사업계획에서 재배분 시도 방지 | FO 대시보드 렌더링 시 `isBusiness` 분기를 통해 재배분 UI 진입점을 완전히 차단. 재배분은 확정된 '운영계획' 탭에서만 수행. |
