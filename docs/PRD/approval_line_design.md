## 1. 기능 개요

교육 신청 시 결재선을 **교육비 총액**(축1)과 **세부산출근거 soft_limit 초과 여부**(축2)의 2축으로 자동 결정하는 시스템.
결재 방식은 **계정(account)별**로 외부결재 / 자체결재 / 통합결재 3단계로 설정 가능.
통합결재 시에만 VOrg 협조처가 결재 노드로 연동된다.

## 2. 사용자 스토리

> "예산 담당자는 BO 계정관리 화면에서 각 계정의 결재 방식(외부/자체/통합)을 설정할 수 있다."
> "예산 담당자는 결재라인 편집 화면에서 금액 구간별 결재 노드(기안/승인/협조/참조)를 설정할 수 있다."
> "예산 담당자는 조건부 협조처 노드에 대해 UI 룰 빌더로 활성화 조건을 설정할 수 있다."
> "학습자가 교육신청서를 제출하면, 총액과 산출근거 항목별 상한 준수 여부에 따라 결재선이 자동 구성된다."

## 4. 핵심 설계

### 4.1 결재 방식 3단계 (계정별 적용)

```
┌──────────────┬──────────────────┬──────────────────────────────┐
│   외부결재    │    자체결재       │        통합결재              │
│  (external)  │  (platform)      │     (integrated)            │
├──────────────┼──────────────────┼──────────────────────────────┤
│ 축1 ❌       │ 축1 ✅           │ 축1 ✅                      │
│ 축2 ❌       │ 축2 ❌           │ 축2 ✅                      │
│ 협조처 ❌    │ 협조처 ❌ 숨김   │ 협조처 ✅                   │
│ 결재선 ❌    │ 결재선 승인만     │ 결재선 전체                 │
└──────────────┴──────────────────┴──────────────────────────────┘
```

### 4.2 2축 에스컬레이션 모델

```
축1: 교육비 신청 총액 → 승인자 레벨 상승
  100만↓: 팀장 전결
  100~500만: 팀장 → 실장
  500만↑: 팀장 → 실장 → 본부장

축2: 세부산출근거 soft_limit 초과 → 조건부 협조처 활성화
  soft 이내: 조건부 협조처 = 참조(CC)
  soft 초과: 조건부 협조처 = 협조(결재필요)
```

### 4.3 시나리오 매트릭스

| | soft 이내 | soft 초과 |
|---|----------|----------|
| **소액** | 팀장 전결 + CC | 팀장 + 협조처 |
| **고액** | 팀장→실장→본부장 + CC | 팀장 + 협조처 → 실장→본부장 |

### 4.4 노드 타입 정의

| 타입 | 코드 | 설명 | 데이터 소스 |
|------|------|------|-------------|
| 기안 | `draft` | 기안자 (자동) | 신청자 본인 |
| 승인 | `approval` | 결재 승인권자 | 직접 입력 (역할명) |
| 협조 | `coop` | 협조처 (병렬) | VOrg coopTeams 자동 참조 |
| 참조 | `reference` | 참조처 (통보) | VOrg coopTeams (soft 미초과 시) |

## 5. DB/데이터 구조

### 5.1 ACCOUNT_MASTER 확장
```js
{ code: 'HMC-OPS', approvalSystem: 'platform',   ... }  // 자체결재
{ code: 'HMC-RND', approvalSystem: 'integrated',  ... }  // 통합결재
{ code: 'HMC-PART', approvalSystem: 'external',   ... }  // 외부결재
```

### 5.2 APPROVAL_ROUTING 노드 구조 (approvers 완전 제거)
```js
ranges: [{
  max: 5000000, label: '100~500만원',
  nodes: [
    { type: 'draft',    label: '기안자', order: 1 },
    { type: 'approval', label: '팀장',  role: '팀장', order: 2 },
    { type: 'coop', coopType: '교육협조처',
      activation: 'conditional', conditionRuleId: 'RULE-001',
      requiresIntegrated: true, order: 3 },
    { type: 'coop', coopType: '재경협조팀',
      activation: 'conditional', conditionRuleId: 'RULE-002',
      requiresIntegrated: true, order: 4 },
    { type: 'approval', label: '실장', role: '실장', final: true, order: 5 },
  ]
}]
```

### 5.3 조건 룰 (APPROVAL_COOP_RULES)
```js
APPROVAL_COOP_RULES = [{
  id: 'RULE-001',
  name: '교육협조처 활성화 조건',
  tenantId: 'HMC',
  operator: 'OR',
  conditions: [
    { field: 'soft_limit_exceeded', op: 'eq', value: true },
    { field: 'edu_type', op: 'in', value: ['해외교육', '학회참석'] },
  ]
}]
```

### 5.4 사용 가능한 조건 필드

| 필드 | 코드 | 타입 | 설명 |
|------|------|------|------|
| 세부산출근거 soft 초과 | `soft_limit_exceeded` | boolean | 1개라도 soft 초과 시 true |
| 교육유형 | `edu_type` | select | 교육유형 목록에서 선택 |
| 신청 총액 | `total_amount` | number | 교육비 합계 금액 |
| 교육기간(일) | `edu_days` | number | 교육 일수 |
| 해외교육 여부 | `is_overseas` | boolean | 해외 교육 여부 |

### 5.5 기존 DB 활용 (스키마 변경 불필요)
```sql
-- calc_grounds 테이블 (이미 존재)
limit_type  TEXT    DEFAULT 'none'    -- 'none' | 'soft' | 'hard' | 'both'
soft_limit  BIGINT  DEFAULT 0
hard_limit  BIGINT  DEFAULT 0
```

## 6. 비즈니스 로직

### 6.1 결재선 생성 알고리즘
```
1. 계정의 approvalSystem 확인
2. 외부결재 → 결재선 생성 안 함 (참고 정보만)
3. 축1: 교육비 총액으로 금액 구간 결정 → 해당 구간의 nodes 로드
4. 노드 필터링:
   - requiresIntegrated && approvalSystem !== 'integrated' → 제외
   - activation === 'conditional' → 룰 평가 (evaluateRule)
5. coop 노드의 실제 팀 = VOrg coopTeams에서 coopType으로 매핑
6. 최종 결재선 반환
```

### 6.2 조건 룰 평가 (evaluateRule)
```
operator === 'OR': 조건 중 하나라도 true → 활성화
operator === 'AND': 모든 조건이 true → 활성화
```

## 8. 예외 처리 및 엣지 케이스

| # | 케이스 | 처리 방식 |
|---|--------|----------|
| 1 | 복수 항목 중 일부만 soft 초과 | 1개라도 초과 시 에스컬레이션 발동 |
| 2 | soft_limit = 0 (미설정) | limit_type='none'이면 soft 검사 스킵 |
| 3 | soft = hard 동일 금액 | 실질 hard-only와 동일. UI 경고 표시 |
| 4 | 에스컬레이션으로 결재선 중복 | 중복 노드 자동 제거 |
| 5 | 금액 수정 후 soft 이하로 변경 | 실시간 재계산, 결재라인 복귀 |
| 6 | VOrg에 협조처 미등록 | coop 노드 스킵 + 관리자 경고 로그 |
| 7 | 총액 경계값 (정확히 100만원) | < 비교, 100만원은 다음 구간. UI 알림 |
| 8 | 결재 방식 미설정 | 기본값 'platform' (자체결재) 적용 |
| 9 | 자체→통합 전환 중 진행건 | 제출 시점의 결재 방식 유지 |
| 10 | 통합→자체 다운그레이드 | 진행 중 협조 결재 건은 관리자 판단 |
| 11 | 관리자의 soft 일괄 변경 | 기존 제출건은 제출 시점 기준 유지 |
| 12 | 외부결재에서 soft 초과 | 에스컬레이션 없음. FO 경고 배너만 표시 |
