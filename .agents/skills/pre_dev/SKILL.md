# Pre-Dev Check 스킬

## 언제 사용하나?
- 기존 JS/CSS 파일을 **수정** 하려고 할 때
- 새로운 기능 개발 전 **파일 크기** 및 **리팩토링 필요 여부** 를 확인하고 싶을 때
- 관련 **PRD, 태스크** 를 빠르게 참고하고 싶을 때

## 실행 방법

```bash
# 수정 예정 파일 지정 (공백으로 여러 개 가능)
node scripts/pre_dev_check.js bo_approval.js plans.js

# 파일 미지정 시 전체 현황만 출력
node scripts/pre_dev_check.js
```

## 출력 항목

| # | 항목 | 설명 |
|---|------|------|
| ① | **수정 대상 파일 크기** | 🔴 100KB↑ = 분리 먼저 / 🟡 70KB↑ = 주의 |
| ② | **전체 JS 크기 현황** | 상위 15개 파일, 리팩토링 후보 표시 |
| ③ | **관련 PRD 파일** | 파일명 키워드로 자동 매칭 |
| ④ | **dev_progress.md 맥락** | 최근 12줄 (이전 개발 흐름) |
| ⑤ | **스킬/워크플로우 가이드** | PRD Manager, auto_deploy 등 리마인더 |
| ⑥ | **즉시 리팩토링 필요 목록** | 100KB 초과 전체 파일 |

## 리팩토링 기준

```
🔴 100KB 이상  →  개발 시작 전 반드시 모듈 분리 후 작업
🟡  70KB 이상  →  분리 검토 권장 (기능 추가 시 우선 고려)
✅  70KB 미만  →  정상 범위, 바로 개발 진행
```

## AI 에이전트 적용 규칙

1. 기존 파일을 수정하는 개발 요청이 오면, **반드시 먼저** 이 스킬을 적용한다.
2. 체크 결과에서 `🔴 REFACTOR 필요` 파일이 있으면:
   - **개발 전에 먼저 모듈 분리**를 진행한다
   - 분리 후 새 파일에 기능을 추가한다
3. 체크 결과를 기반으로 관련 PRD를 참조하여 요구사항 정합성을 확인한다
4. `dev_progress.md` 맥락을 참고하여 이전 세션 작업과의 연속성을 유지한다
