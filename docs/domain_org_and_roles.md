# 도메인: 가상교육조직 및 역할 매핑 (Org & Roles Domain)

## 1. 개요
백오피스의 사용자 역할(Role) 체계와 회사의 각 부서, 그리고 가상(Virtual) 단위로 묶인 예산 관리 조직을 연동하는 심장 역할을 하는 도메인 지식입니다. (`bo_roles.js` 와 `bo_virtual_org_unified.js` 간의 연결)

## 2. 권한 계층 트리 (Role Hierarchy)
권한은 최상단 **테넌트(Tenant) 기반으로 완벽히 분리**되어 있으며, 기본적으로 세 단계의 거버넌스로 이어집니다.
1. `platform_admin`: 시스템 전체를 조망하는 HMG Super Admin.
2. `(Company)_tenant_admin`: HMC, KIA 등 각 자사(테넌트) 레벨의 최고 예산/기획 권한.
3. `head`: 예산 총괄관리자 (예: HMC 일반예산 총괄). 큰 예산 풀의 주도권을 갖습니다.
4. `ops`: 예산 운영담당자 (예: HMC 지역 센터 담당 운영진). 총괄관리자 밑에서 하위 예산들의 징수/실행을 검증합니다.

## 3. 매핑 로직의 핵심: 리버스 맵핑 (Reverse Sync)

*   **기존 로직 (과거 방식):**
    과거에는 역할(`roles`) 탭에서 먼저 사용자를 검색하여 추가해 주어야, 차후 제도(가상조직 템플릿) 설정 시 해당 보유자만 목록에서 필터링되어 고를 수 있었습니다. (절차가 2중으로 분리됨)

*   **강화된 로직 (현재 방식):**
    제도 설정 화면에서 **테넌트 전체 사용자**를 조회/검색할 수 있습니다. 
    그리고 이 중에서 총괄/운영담당자를 지정(맵핑)하는 즉시, `user_roles` (역할 DB 테이블)에 Supabase Upsert 쿼리가 날아갑니다. **즉, 제도 화면을 세팅하는 행위 하나만으로 권한이 자동으로 서버에 배정(Reverse Sync)** 됩니다.

## 4. 권한 부여 유효기간 관리 (Role Duration Config)
현 시스템은 **`user_roles`** DB 테이블 상에 `start_date`, `end_date` 컬럼을 둠으로써 직무 발령 기간에 맞춘 권한 기한 제어(`Role Lifecycle`)를 지원합니다.
- 매핑과 동시에 필수적으로 `종료일`을 입력받습니다.
- 총괄 또는 운영담당자 라인에서 "삭제(- 닫기)" 처리할 경우 가상조직만 빠지는 것이 아니라, 백엔드 테이블에서도 `DELETE` 쿼리가 날아가 부여받았던 **역할(Role) 정보 자체를 깔끔하게 자동 회수**합니다.
