## 1. 정책 정의 (핵심)

### "배정"과 "통장 입금"은 별개 행위

```
[배정(Allocation)]
  = 총괄담당자가 "이 교육에 이만큼 쓸 수 있다"고 승인한 것
  = plans.allocated_amount에 기록
  = 약속/가이드라인이지 실제 돈이 입금된 것이 아님

[통장 입금(Deposit)]
  = 나중에 별도로 팀 통장에 실제 예산을 넣어주는 행위
  = bankbook.balance에 반영
  = 이 돈이 있어야 실제 교육 신청이 가능
```

### 2가지 운영 시나리오 (모두 지원 필요)

#### 시나리오 A: 계획 선행 → 배정 → 통장 입금
```
[Step 1] FO 사용자 → 교육계획 제출 (100만, 200만)
    ↓
[Step 2] BO 총괄담당자 → 배정 결정 (80만, 160만) ← 가이드라인
    ↓  ※ 돈은 아직 움직이지 않음
[Step 3] BO 총괄담당자 → 팀 통장에 240만 입금 (별도 행위)
    ↓
[Step 4] FO 사용자 → 배정 현황 + 통장 잔액 확인
    ↓
[Step 5] FO 사용자 → 재배분 (80→100, 160→0)
    ↓
[Step 6] FO 사용자 → 100만원 교육 신청 (통장 차감)
    ↓
[Step 7] FO 사용자 → 잔여 140만으로 수시 계획 → 신청
```

#### 시나리오 B: 통장 선입금 → 수시 계획 후행
```
[Step 1] BO 총괄담당자 → 팀 통장에 미리 500만 입금
    ↓  ※ 교육계획 없이 통장만 먼저 충전
[Step 2] FO 사용자 → 수시 교육계획 100만원 제출
    ↓
[Step 3] 승인 프로세스 → 교육계획 승인
    ↓  ★ 승인 시 allocated_amount = amount 자동 확정
[Step 4] FO 사용자 → 승인된 계획으로 교육 신청 (통장 차감)
    ↓
[반복] 통장 잔액 내에서 수시 계획 → 승인 → 신청 반복
```

### 배정의 이중 제약 조건

```
교육 신청이 가능하려면 다음 2가지 모두 충족:
  ① 해당 교육계획에 배정액이 있음 (plans.allocated_amount > 0)
  ② 팀 통장에 잔액이 있음 (bankbook.balance ≥ 신청 금액)

※ 배정액과 신청액은 다를 수 있음
  - 배정 80만인 교육에 100만원 신청 → 통장 잔액 충분하면 허용
  - 단, 배정 0원인 교육 → 신청 불가 (배정이 없으면 사용 권한 없음)

※ 배정이 설정되는 2가지 경로:
  경로 1: BO 총괄담당자가 직접 할당 (시나리오 A)
  경로 2: 교육계획 승인 시 자동 확정 — allocated_amount = amount (시나리오 B)
```

## 2. 시나리오 재시뮬레이션

```
[Step 1] 교육계획 제출
  plans: { id: A, amount: 1,000,000 }, { id: B, amount: 2,000,000 }
  통장: 없음
  → 현재: ✅ 동작

[Step 2] BO 배정 결정
  plans.A.allocated_amount = 800,000
  plans.B.allocated_amount = 1,600,000
  통장: 변동 없음 (아직 입금 안 함)
  → 현재: ✅ 동작 (인라인 편집 또는 시뮬레이션 확정)

[Step 3] BO 팀 통장 입금 (240만원)
  bankbook.balance += 2,400,000
  → 현재: 🟡 bo_allocation.js에서 수동 가능
  → 문제: 배정 합계 기준으로 자동 입금하는 기능 없음
  → 필요: "배정 기반 통장 입금" 편의 기능

[Step 4] FO 배정+통장 확인
  → 현재: 🔴 FAIL — FO에 배정액/통장 잔액 미표시

[Step 5] FO 배정 재배분 (80→100, 160→0)
  plans.A.allocated_amount = 1,000,000
  plans.B.allocated_amount = 0
  → 현재: 🔴 FAIL — FO 재배분 UI 없음

[Step 6] 교육A 신청 (100만원)
  application 생성 + bankbook.balance -= 1,000,000
  잔액: 1,400,000
  → 현재: 🔴 FAIL — 통장 잔액 검증/차감 없음, plan_id 연결 없음

[Step 7] 수시 교육계획 140만원 → 교육 신청
  plans.C { amount: 1,400,000, allocated_amount: 1,400,000 }
  bankbook.balance -= 1,400,000 → 0원
  → 현재: 🔴 FAIL — 수시 계획 잔액 검증 없음
```

**결과: Step 1-2 통과, Step 3 부분 가능, Step 4-7 FAIL**

## 3. 갭 분석 (수정)

### 🟡 GAP-01: BO 배정 → 통장 입금 편의 기능

| 항목 | 현재 | 필요 |
|------|------|------|
| 통장 입금 | bo_allocation에서 수동 가능 | "배정 합계 기준 일괄 입금" 버튼 편의 기능 추가 |
| 통장 자동 생성 | 수동 생성 | 해당 팀+계정 통장 없으면 자동 생성 |

> ※ 배정 = 통장 입금이 아니므로, 자동 입금은 강제가 아니라 "편의 기능"임

---

### 🔴 GAP-02: FO 통장 잔액/배정 현황 표시

| 항목 | 현재 | 필요 |
|------|------|------|
| FO 대시보드 | 통장 잔액 미표시 | 내 팀 통장 잔액 카드 |
| FO 교육계획 리스트 | 배정액 미표시 | allocated_amount 컬럼 표시 |
| FO 교육계획 상세 | 배정액 미표시 | 배정액 + 통장 잔액 표시 |

---

### 🔴 GAP-03: FO 사용자 배정 재배분 UI

| 항목 | 현재 | 필요 |
|------|------|------|
| 재배분 UI | ❌ 없음 | FO 교육계획 리스트에서 배정액 인라인 편집 |
| 제약 조건 | 없음 | Σ재배분 ≤ 통장 잔액 (배정 합이 통장보다 크면 안 됨) |
| 배정 0원 처리 | 없음 | 배정 0원 = 해당 교육 포기 (신청 불가) |

---

### 🔴 GAP-04: FO 교육신청 시 이중 검증

| 항목 | 현재 | 필요 |
|------|------|------|
| 배정 검증 | ❌ 없음 | allocated_amount > 0 인 교육만 신청 가능 |
| 통장 잔액 검증 | ❌ 없음 | bankbook.balance ≥ 신청금액 |
| 통장 차감 | ❌ 없음 | 신청 승인 시 통장에서 차감 |
| 반려 시 환불 | ❌ 없음 | 반려 시 통장에 복원 |

---

### 🔴 GAP-05: application ↔ plan 연결

| 항목 | 현재 | 필요 |
|------|------|------|
| plan_id 연결 | ❌ 미연동 | 교육신청 생성 시 반드시 plan_id 연결 |
| 교육계획 → 신청 동선 | ❌ 없음 | FO 교육계획에서 [교육 신청] 버튼 |
| 중복 방지 | ❌ 없음 | 같은 plan_id로 중복 진행 중 신청 방지 |

---

### 🟡 GAP-06: 수시 교육계획 + 통장 연동

| 항목 | 현재 | 필요 |
|------|------|------|
| 수시 계획 금액 검증 | ❌ 없음 | 통장 잔액 초과 시 경고 (차단은 아님) |
| 수시 승인 = 배정 확정 | ❌ 없음 | 수시 계획 승인 시 자동으로 allocated_amount 확정 |

---

### 🔴 GAP-07: 교육계획 승인 시 자동 배정 (시나리오 B 필수)

| 항목 | 현재 | 필요 |
|------|------|------|
| 승인 시 allocated_amount 세팅 | ❌ 없음 | 승인 시 allocated_amount = amount 자동 설정 |
| BO 직접 배정과 공존 | ❌ 없음 | BO가 미리 배정했으면 그 값 유지, 없으면 자동 확정 |
| 승인 시 통장 잔액 확인 | ❌ 없음 | 승인 시점에 통장 잔액 ≥ amount 검증 (경고) |

## 4. 엣지 케이스 (비판적 검증)

| # | 케이스 | 처리 방식 |
|---|--------|----------|
| EC-01 | 배정 80만인데 100만 신청 | ✅ 허용 — 통장 잔액이 충분하면 OK (배정은 가이드라인) |
| EC-02 | 배정 0원인 교육에서 신청 시도 | 🔴 차단 — 배정 없으면 신청 권한 없음 |
| EC-03 | 통장 잔액 부족 상태에서 신청 | 🔴 차단 — 잔액 ≥ 신청액 필수 |
| EC-04 | 재배분 합 > 통장 잔액 | 🔴 차단 — 재배분 합이 통장을 초과할 수 없음 |
| EC-05 | 이미 신청한 교육의 배정을 0원으로 변경 | 🔴 차단 — 진행 중 신청이 있으면 배정 삭감 불가 |
| EC-06 | 동시에 2명이 같은 통장에서 신청 | 🟡 optimistic lock — 잔액 재확인 후 처리 |
| EC-07 | 통장 미입금 상태에서 교육 신청 | 🔴 차단 — 통장 잔액 0이면 신청 불가 |
| EC-08 | 교육 반려 → 통장 환불 | ✅ 자동 환불 — bankbook 잔액 복원 |
| EC-09 | 수시 교육계획 금액 > 통장 잔액 | 🟡 경고만 — 나중에 추가 입금 받을 수 있으므로 차단하지 않음 |
| EC-10 | 배정 재배분 후 BO에서 배정 변경 | 🟡 BO 배정이 우선 — FO 재배분은 덮어쓰기됨 (알림 필요) |
| EC-11 | 통장 선입금 후 교육계획 없이 바로 교육 신청 | 🔴 차단 — 반드시 교육계획 승인 후에만 신청 가능 |
| EC-12 | BO가 미리 배정(80만) 했는데 승인 시 자동배정(100만)으로 덮어쓰기 | 🟡 BO 배정이 이미 있으면 자동배정 스킵 — BO 값 우선 |
| EC-13 | 시나리오 A와 B가 혼합 (일부 계획은 BO 배정, 일부는 수시 자동배정) | ✅ 허용 — 통장 잔액이 총 제약, 개별 배정은 경로 무관 |

## 5. 개발 Phase (수정)

| 순서 | Phase | 내용 | 수정 파일 | 규모 |
|------|-------|------|----------|------|
| 1 | **A: FO 배정+통장 표시** | FO 교육계획에 배정액/통장잔액 표시 | `plans.js`, `dashboard.js` | 중 |
| 2 | **B: FO 배정 재배분** | FO 인라인 편집 + 합계≤통장 검증 | `plans.js` | 대 |
| 3 | **C: application↔plan 연결** | plan_id 세팅 + 교육계획→신청 동선 | `plans.js`, `apply.js` | 중 |
| 4 | **D: 교육신청 통장 검증** | 배정>0 검증 + 통장잔액 검증 + 차감/환불 | `apply.js`, `bo_approval.js` | 대 |
| 5 | **E: BO 배정→통장 편의기능** | 배정 합계 기준 일괄 입금 버튼 | `bo_plan_mgmt.js` | 소 |
| 6 | **F: 수시 계획 통장 연동** | 수시 금액 경고 + 승인 시 자동 배정 | `plans.js`, `bo_approval.js` | 중 |
| 7 | **G: 교육계획 승인→자동 배정** | 승인 시 allocated_amount 자동 확정 (BO 미배정 시) | `bo_approval.js` | 소 |
