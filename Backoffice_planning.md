## 1. 예산 배정 아키텍처 (Budget Source Architecture)

| 유형 | 대상 계정 | 관리 단위 | 담당자 |
|------|---------|---------|--------|
| **🔗 SAP I/F 연동형** | HMC-OPS, HMC-ETC, HMC-PART, KIA-OPS, KIA-PART | 가상본부 단위 자동 매핑 | 신승남, 고범현 |
| **✏️ 플랫폼 자체 관리형** | HMC-RND, HAE-OPS, HAE-PART, HAE-CERT | 계정/센터 단위 직접 입력 | 류해령, 안슬기 |

### SAP I/F 연동형 프로세스
```
SAP ERP → I/F Hub → 가상본부 단위 자동 배정 → 담당자 정합성 확인·확정 → 팀 배분
```
- 수신 실패 시 **다시 불러오기(배치 재전송)** 기능 제공
- 수동 수정 제한 — 증액은 **추가 배정** 기능으로만 허용

### 플랫폼 자체 관리형 프로세스
```
총괄 담당자 직접 입력 → 확정 → 하위 팀 즉시 사용 가능
```
- 연중 증액 시 **추가 배정** 팝업으로 금액 가산 + 이력 자동 저장

## 2. 예산 원장 공식

```
현재 잔액 = 기초배정액 + 추가배정 누계 − 가점유액 − 실지출액
```

| 구분 | 의미 |
|------|------|
| 기초배정 | SAP I/F 수신 또는 직접 입력 연간 총액 |
| 추가배정 | 연중 증액 처리 누계 |
| 가점유 | 신청 후 승인 전 임시 점유 금액 |
| 실지출 | 승인 확정 후 실제 집행 금액 |

### 소진율 임계치 알림
- 🟢 80% 미만: 정상
- 🟡 80% 이상: 주의
- 🔴 95% 이상: 위험

## 3. 주요 기능 (4탭 구조)

| 탭 | 기능 | 처리 권한 |
|----|------|---------|
| 📊 배정 현황 원장 | 계정별 원장 공식 + 소진율 + 조직별 상세 | 오너 + 위임된 담당자 |
| ➕ 추가 배정 | 연중 증액 처리 + Audit Trail 자동 기록 | **계정 오너만** |
| ↔ 예산 이관 | 동일 계정 내 A조직→B조직 이관 + 사유 필수 | **계정 오너만** |
| 📋 변경 이력 | 전체/조직별 Audit Trail (SAP수신·확정·추가·이관) | 오너 + 위임된 담당자 |

## 4. 데이터 모델 (bo_data.js)

### BUDGET_SOURCE_TYPE
계정코드 → 재원 유형(`sap_if` / `platform`) 매핑

### BUDGET_ALLOCATIONS
```javascript
{ id, tenantId, accountCode, org, orgType,
  baseAmount, addAmount, reserved, spent,
  sourceType, (ifReceivedAt|enteredAt), (confirmedBy|enteredBy), status }
```

### ALLOCATION_HISTORY (Audit Trail)
```javascript
{ id, allocationId, date, type, amount, note, by }
// type: 'SAP_IF' | '기초입력' | '확정' | '추가배정' | '이관출처' | '이관수신'
```
