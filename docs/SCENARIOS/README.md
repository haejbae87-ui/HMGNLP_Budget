# 📋 비즈니스 시나리오 관리 레지스트리

> **목적**: 사용자가 제안한 업무 시나리오를 체계적으로 관리하고, 개발 완료 후 검증에 활용
> **원칙**: 시나리오 그대로 동작하지 않더라도, **업무 목적을 달성할 수 있는 경로**가 존재하면 ✅

---

## 상태 정의

| 상태 | 의미 |
|------|------|
| 🔴 **불가능** | 현재 설계/구현에서 업무 목적을 달성할 수 없음 |
| 🟡 **설계 완료** | PRD에 반영되었으나 코드 미구현 |
| 🟢 **구현 완료** | 코드 구현 완료, 수동 검증 대기 |
| ✅ **검증 완료** | 브라우저 테스트 등으로 시나리오 동작 확인 |
| 🔄 **대안 경로** | 원래 시나리오와 다르지만 업무 목적 달성 가능 |

## 파일 명명 규칙

```
SC-{3자리 번호}_{영문_snake_case}.md
예: SC-001_plan_budget_reallocation.md
```

## 시나리오 파일 템플릿

```markdown

> **제안일**: YYYY-MM-DD
> **상태**: 🔴/🟡/🟢/✅/🔄
> **업무 목적**: [이 시나리오로 달성하려는 핵심 업무 목적 1줄]
> **관련 PRD**: [링크]

## 시나리오 스텝

1. 사용자가 ...
2. ...

## 분석 결과

### 현재 가능 여부: ✅/❌
- GAP-1: ...

### 필요한 변경사항
- ...

## 대안 경로 (있는 경우)

> 시나리오 스텝과 다르지만 업무 목적을 달성할 수 있는 방법

## 검증 체크리스트

- [ ] Step 1: ...
- [ ] Step 2: ...
```
