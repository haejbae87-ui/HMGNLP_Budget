## 1. 기능 개요

현재 패턴A(계획→신청→결과) 교육계획 연동은 **개인직무 사외학습 > R&D 예산 계정**에만 구현되어 있다. 
그러나 **교육운영 목적**(이러닝/집합 운영, 워크샵/세미나/콘퍼런스 운영, 기타 운영)에서도 패턴A를 사용하는 정책이 설정될 수 있으며, 이때 교육계획 선택이 필수이다.

### 문제점 (As-Is)

| 구분 | 개인직무사외학습 (R&D) | 교육운영 (패턴A) |
|---|---|---|
| 교육계획 선택 UI | ✅ 인라인 카드 목록 | ❌ 없음 (프로세스 안내 배너만 표시) |
| 교육계획 없을 때 | ✅ 경고 + 계획 수립 바로가기 | ❌ 예산 선택 후 다음 진행 가능 (오류) |
| 복수 선택 | ❌ 단건+체크박스 혼용 | ❌ 미지원 |
| 교육계획 표시 | 인라인 (화면 길어짐) | 없음 |

### 핵심 리스크

> [!CAUTION]
> 교육운영 패턴A에서 교육계획 없이 신청이 진행되면, BO에서 계획-신청-결과 연계가 깨지며 예산 추적이 불가해진다.

## 2. 사용자 스토리

> "교육담당자(피플육성팀 박보검)는 교육신청 → 예산 선택 단계에서 운영계정(패턴A)을 선택할 때, 사전에 승인받은 교육계획을 팝업에서 복수 선택하여 연동할 수 있다."

> "교육담당자가 패턴A 운영계정을 선택했으나 승인된 교육계획이 없으면, '교육계획이 없습니다' 경고와 함께 교육계획 수립 바로가기가 표시되어 신청 진행이 차단된다."

## 3. 상세 기능 요구사항

| 번호 | 기능 | 설명 | 우선순위 |
|---|---|---|---|
| F-001 | 교육운영 패턴A 계획 필수화 | 교육운영 목적(이러닝/집합, 워크샵/세미나, 기타 운영)에서 패턴A 정책 매칭 시, Step 2에서 교육계획 선택을 필수로 요구 | 🔴 HIGH |
| F-002 | 교육계획 선택 팝업 | 교육계획 목록을 팝업(모달)로 표시, 검색/필터 지원, 복수 선택 가능. **R&D 포함 전 목적 통합** | 🔴 HIGH |
| F-003 | 교육계획 미존재 시 차단 | 승인된 교육계획이 0건이면 경고 문구 + 교육계획 수립 바로가기 표시, 다음 버튼 비활성화 | 🔴 HIGH |
| F-004 | 선택된 계획 요약 표시 | 팝업 확인 후, Step 2에 선택된 교육계획 카드(건수, 예산 합계) 요약 표시 | 🟡 MEDIUM |
| F-005 | R&D 인라인 → 팝업 전환 | 개인직무사외학습 R&D 계획 선택도 동일 팝업 컴포넌트로 교체 (인라인 제거) | 🔴 HIGH |
| F-006 | 교육유형 자동 적용 | 패턴A에서 교육계획 선택 시, 계획에 포함된 교육유형을 Step 3에 자동 반영하여 중복 선택 방지 | 🟡 MEDIUM |
| F-007 | 교육계획 팀 레벨 공유 | 같은 팀에서 작성한 승인된 교육계획 전부 노출 (팀 공유 기본값) | 🔴 HIGH |
| F-008 | 교육계획 노출 범위 설정 (향후) | 정책/설정에서 "본인 승인 계획만" vs "팀 전체 승인 계획" 선택 옵션 추가 | 🟢 LOW |

## 4. 교육계획 선택 팝업 설계

### 4.1 트리거 조건

```
사용자가 Step 2에서 예산 계정을 선택 →
  getProcessPatternInfo()가 패턴 'A' 반환 →
    해당 예산계정의 budgetId에 매칭되는 승인된 교육계획 조회 →
      0건: 경고 UI 표시 + 다음 차단
      1건 이상: "📋 교육계획 선택" 버튼 표시 → 클릭 시 팝업 오픈
```

### 4.2 팝업 UI 명세

```
┌─────────────────────────────────────────────┐
│  📋 승인된 교육계획 선택       (검색 필터)     │
│                                  ✕ 닫기      │
├─────────────────────────────────────────────┤
│  🔍 검색: [________교육계획명 검색________]    │
│                                              │
│  ☐  내구기술 이러닝 계획_260403_2             │
│     📅 2026-04-03  💰 예산 500,000원          │
│     ✅ 잔액 500,000원                         │
│                                              │
│  ☑  피플육성 집합교육 Q2 계획                  │
│     📅 2026-03-15  💰 예산 2,000,000원        │
│     ✅ 잔액 1,500,000원                       │
│                                              │
│  ☐  리더십 워크숍 연간 계획                    │
│     📅 2026-01-20  💰 예산 5,000,000원        │
│     ⚠️ 잔액 부족                              │
│                                              │
├─────────────────────────────────────────────┤
│  선택됨: 1건 | 총 예산: 2,000,000원           │
│                               [확인] [취소]   │
└─────────────────────────────────────────────┘
```

### 4.3 팝업 동작 규칙

| 항목 | 규칙 |
|---|---|
| 선택 방식 | 체크박스 복수 선택 |
| 정렬 | 최신 계획 우선 (생성일 DESC) |
| 필터 | 계획명 텍스트 검색 |
| 잔액 표시 | 잔액 > 0: 녹색, 잔액 ≤ 0: 빨간색 + "잔액 부족" |
| 기간 만료 | 만료 계획은 선택 불가 (opacity:0.5 + "기간만료" 태그) |
| 확인 | 1건 이상 선택해야 확인 버튼 활성 |
| 취소 | 기존 선택 유지 |

## 5. 비즈니스 로직

### 5.1 패턴A 교육계획 필수화 로직

```javascript
// Step 2 예산 선택 후 프로세스 정보 조회
const _processInfo = getProcessPatternInfo(persona, purposeId, accountCode);

if (_processInfo?.pattern === 'A') {
  // 해당 예산의 승인된 교육계획 조회 
  const plans = _dbApprovedPlans.filter(p => p.budgetId === selectedBudgetId);
  
  if (plans.length === 0) {
    // 경고 표시 + 다음 버튼 비활성
    showNoPlanWarning();
  } else {
    // "교육계획 선택" 버튼 표시
    showPlanPickerButton();
  }
}
```

### 5.2 교육계획 매칭 기준

| 조건 | 매칭 기준 |
|---|---|
| 예산 계정 | `plan.budgetId === selectedBudgetId` 또는 같은 계정 유형 |
| 팀 소속 | 같은 팀(persona.team)에서 작성한 교육계획 전부 노출 |
| 승인 상태 | `plan.status === 'approved'` (DB: `plan_status`) |
| 유효 기간 | `plan.endDate >= today()` 또는 endDate 없음 |
| 잔액 | 표시만 (잔액 0도 선택 가능, 경고만 표시) |

## 7. 예외 처리 및 엣지 케이스

| 케이스 | 처리 방식 |
|---|---|
| 교육계획 0건 | 경고 배너 + 교육계획 수립 바로가기 + 다음 차단 |
| 교육계획 선택 후 예산 변경 | 선택된 계획 초기화 + 재선택 필요 |
| 교육계획 잔액 부족 | 선택 가능하되 "잔액 부족" 경고 표시 |
| 교육계획 기간 만료 | 선택 불가 + "기간만료" 태그 + 비활성 처리 |
| 복수 계획의 교육유형 상충 | Step 3에서 병합된(union) 교육유형 표시 |
| R&D 계정 교육계획 | 팝업 방식으로 통합 (기존 인라인 제거) |

## 8. 기획자 검토 필요 항목

> [!IMPORTANT]
> **교육운영 목적에서 패턴A 정책이 설정되어 있는데 승인된 교육계획이 없는 경우**, 현재는 아무 경고 없이 신청이 진행됩니다. 이는 예산 통제 정책에 위배됩니다.

> [!WARNING]
> **개인직무사외학습 R&D 인라인 목록**: 현재 교육계획이 많아지면 Step 2 화면이 과도하게 길어집니다. 팝업 방식으로 통합하는 것이 UX 일관성 면에서 바람직하나, 기존 사용자 학습 비용이 발생할 수 있습니다.

> [!NOTE]
> **교육운영의 예산 계정별 교육계획 필터링**: `_dbApprovedPlans`에서 `budgetId`로 필터링하는데, 교육운영 계정(운영계정 HMC-OPS)은 여러 팀의 계획이 공유될 수 있습니다. 담당자별/팀별 추가 필터가 필요할지 검토 필요.
